How to use Bangla Keyboard

  1. 1
    Switch Layouts: Press Super + Space (Windows Key + Space) to toggle input source between English (US) and Bangla (Phonetic).
  2. 2
    Visual Indicator: GNOME's top right header status bar will change from en to বাং to indicate active Bangla typing.
  3. 3
    Start Typing: Type phonetically (e.g., type bangla to get বাংলা). The smart suggestions display will help autocomplete.

Keyboard Hotkey:

Super + Spacebar

Live USB Persistence Configuration

Learn how to flash the Agami OS ISO to a flash drive and enable persistence to save files, browser histories, and installed software.

What is Live Persistence?

Usually, running an OS from a Live USB is read-only. Any file you download, document you write, or wallpaper setting you change will disappear after rebooting. Persistence creates a separate writeable storage partition on your USB flash drive. This allows the system to save changes securely across sessions.

Benefits for Schools & Labs:

  • Download and retain learning materials offline.
  • Install new programming languages or STEM kits permanently.
  • Saves student settings, languages, and custom desktop shortcuts.

Easy Flashing Options

Flashing with Rufus (Recommended)

Rufus is a lightweight, high-performance Windows tool that natively configures Live Persistence partitions.

  1. Insert a 16GB+ USB Flash Drive. Launch **Rufus**.
  2. Select the `Agami_OS_Education.iso` in Boot Selection.
  3. Important Slider: Set the **Persistent Partition Size** slider (recommend 4GB to 12GB depending on flash size).
  4. Select **FAT32** or **NTFS** and click **START**.

Flashing with Ventoy

Ventoy allows copy-pasting multiple ISOs on a single USB. It requires generating a configuration file for persistence.

  1. Download **Ventoy** and install it on your USB drive.
  2. Download the persistence file template persistence.dat from Ventoy's official plugin page.
  3. Create a directory named `ventoy` inside the root of your USB flash drive and place a file named `ventoy.json` with the following configuration:
{
    "persistence": [
        {
            "image": "/Agami_OS_Education.iso",
            "backend": "/persistence.dat"
        }
    ]
}
